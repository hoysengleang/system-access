1. PROJECT MISSION
============================================================
Build a stable Python package and API foundation for System Access
with reliable setup, clear dependency management, and repeatable
team workflow.

Success Criteria:
- Package metadata is valid and maintainable.
- Local install works in one chosen virtual environment.
- API server starts consistently with uvicorn.
- Team members can onboard from one document.


============================================================
2. CURRENT SCOPE
============================================================
In Scope:
- Package structure under systemAccess/
- Project metadata in pyproject.toml
- Runtime dependencies in docs/dependency/requirement.txt
- API entrypoint in systemAccess/__main__.py

Out of Scope (Now):
- Full authentication logic
- Production deployment pipeline
- Large database integration


============================================================
3. ARCHITECTURE SNAPSHOT
============================================================
Main Folders:
- systemAccess/                         Python package modules
- tests/                                test files
- docs/                                 project documentation
- docs/dependency/requirement.txt       pip dependency list

Minimal Runtime Flow:
1) Activate virtual environment (venv recommended)
2) Install package and dependencies
3) Run API with uvicorn
4) Run tests and fix regressions


============================================================
4. ENVIRONMENT SETUP
============================================================
Standard:
- Keep only one environment: venv/
- Avoid switching between venv and .venv in one project

Setup Commands:
python3 -m venv venv
source venv/bin/activate
pip install --upgrade pip
pip install -r docs/dependency/requirement.txt
pip install -e .

Run API:
uvicorn systemAccess.__main__:app --port 1111 --reload

5. DEPENDENCY BOOK
============================================================
Current dependency list:
- fastapi
- uvicorn
- pydantic
- python-dotenv>=1.0.1,<2.0.0
- typing-extensions>=4.10.0,<5.0.0

Dependency policy:
1) Add package to docs/dependency/requirement.txt
2) If runtime dependency, add it to pyproject.toml
3) Install and test locally
4) Log major dependency issues in troubleshooting section

9. TROUBLESHOOTING LOG
============================================================
Use this log format:

Example:

Interpreter mismatch | Select venv interpreter and reload editor |
Keep one virtual environment

YYYY-MM-DD | uvicorn __main__:app failed |
Wrong module path | Use uvicorn systemAccess.__main__:app |
